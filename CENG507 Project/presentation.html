MoC #1: Synchronous Dataflow (SDF)

Deterministic signal processing pipeline

Why SDF?

  • Fixed sampling rates (2000 Hz)
  • Compile-time guarantees
  • Bounded memory usage
  • Multi-rate processing

The Pipeline

EMG Acquire ‚Üí Bandpass ‚Üí Notch ‚Üí Spatial Filter ‚Üí Decimator ‚Üí Window Buffer ‚Üí Features ‚Üí Normalize ‚Üí Classifier

Verified Properties

‚úì Deadlock-free
‚úì Bounded buffers (800 samples max)
‚úì Latency <50ms guaranteed

MoC #2: Statecharts

Hierarchical control logic that scales

Why Statecharts?

  • Reduces 96+ states down to 20
  • Orthogonal regions for parallel modes
  • History mechanism for smart mode switching
  • Behavioral inheritance

Region 1: System Mode

Initialization ‚Üí Operational ‚Üí Calibration ‚Üí Emergency Stop

Region 2: Grip State

Open Hand | Power Grips | Precision Grips | Gesture Poses

Key Features: Deep history, entry/exit actions, guard conditions

MoC #3: Petri Nets

Concurrent resource management without deadlocks

Why Petri Nets?

  • Model concurrent processes explicitly
  • Represent shared resources
  • Formal deadlock analysis
  • Mutex enforcement

Resources Managed

  • CPU (classifier vs feedback controller)
  • System Bus (EMG acquisition vs motor commands)
  • Motor controllers
  • Stimulation hardware

Formally Verified: ‚úì Bounded ‚úì Live ‚úì Deadlock-free ‚úì Mutual exclusion

Hardware/Software Partitioning

Smart division of responsibilities

Hardware (STM32H7 Cortex-M7)

  • 64-channel ADC with DMA for EMG
  • Hardware FPU for real-time filtering
  • DSP extensions for SIMD processing
  • PWM timers for 5 motor channels
  • DAC for feedback stimulation

Software (FreeRTOS + Libraries)

  • CMSIS-DSP for signal processing
  • TensorFlow Lite Micro for ML inference
  • Task scheduling and communication
  • CAN bus motor control

Why This Split? Hardware handles timing-critical tasks, software provides flexibility

‚úì Solution: INT8 quantization + SIMD optimization
